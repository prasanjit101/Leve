# Security Policy

## Supported Versions

Currently, security updates are actively provided for the following versions of Leve:

| Version | Supported          |
| ------- | ------------------ |
| < 1.0.x | :white_check_mark: |

As Leve is in active development, we recommend always running the latest release.

## Reporting a Vulnerability

Because Leve is a framework that handles authentication, credentials, and sandboxed compute, we take security reports seriously.

**Do not open public GitHub issues for security vulnerabilities.**

Instead, please report security vulnerabilities privately:
1. Send an email to **prasanjitdutta45@gmail.com**.
2. Include a detailed description of the vulnerability, steps to reproduce, and any proof-of-concept (PoC) code.
3. We will acknowledge receipt of your vulnerability report within 48 hours and work to provide a fix as quickly as possible.

We request that you practice responsible disclosure and give us reasonable time to investigate and fix the issue before making it public.
